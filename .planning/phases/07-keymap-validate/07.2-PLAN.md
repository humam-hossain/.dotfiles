# Phase 7.2: Fix Window Picker Keymaps

**Goal:** Add `<leader>ww` (cycle) and `<leader>wm` (picker) window navigation

**Gap Closure:** Closes UAT gap from Phase 7 keymap validation

---

## Tasks

### 7.2.1: Add `<leader>ww` (Cycle Windows)

**File:** `.config/nvim/lua/core/keymaps/registry.lua`

Add to `M.global` section under Window domain (after line 157):

```lua
  {
    id = "window.cycle",
    lhs = "<leader>ww",
    mode = "n",
    desc = "Cycle to next window",
    domain = "w",
    scope = "global",
    action = "<C-w>w",
    opts = { noremap = true, silent = true },
  },
```

### 7.2.2: Add `<leader>wm` (Pick Window)

**File:** `.config/nvim/lua/core/keymaps/registry.lua`

Add to `M.lazy` section under Window domain:

```lua
  {
    id = "window.picker",
    lhs = "<leader>wm",
    mode = "n",
    desc = "Pick a window to switch to",
    domain = "w",
    scope = "lazy",
    plugin = "s1n7ax/nvim-window-picker",
    action = function()
      require("window-picker").pick_window()
    end,
  },
```

### 7.2.3: Verify Keymaps

Run `nvim` and confirm:
- `<leader>ww` cycles through windows
- `<leader>wm` opens window picker
