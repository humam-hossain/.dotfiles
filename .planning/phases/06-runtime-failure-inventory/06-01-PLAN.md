  <action>
Create failure audit script per D-03 through D-22:

1. Shebang: `#!/usr/bin/env bash`
2. Set `set -euo pipefail` (same as nvim-validate.sh)
3. Define vars:
   - `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"`
   - `REPO_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"`
   - `OUTPUT_DIR="$REPO_ROOT/.planning/phases/06-runtime-failure-inventory"`
4. Require jq: `command -v jq >/dev/null || { echo "jq required"; exit 1; }`

5. Call nvim-validate.sh internally (D-04):
   ```
   "$SCRIPT_DIR/nvim-validate.sh" all
   ```

6. Parse health.json for failures:
   - Failed plugins: `jq -r '.plugins[] | select(.loaded == false) | "\(.name): \(.error)"'`
   - Missing tools: `jq -r '.tools[] | select(.available == false) | .name'`
   - Parse startup.log, sync.log, smoke.log for Error/E5108/E484 patterns

7. TODO/FIXME scan (D-06):
   ```
   find "$REPO_ROOT/.config/nvim" -name "*.lua" -exec grep -Hn "TODO\|FIXME\|XXX\|BUG" {} \;
   ```
   Output format: `BUG|todo|<file_path>:<line>: <text>`

8. Git log scan (D-07):
   ```
   git --no-merges --grep="bug" --grep="fix" --grep="error" --grep="crash" ...
   ```
   Use `--no-merges` to avoid merge commit noise.

9. Deduplicate by (description + owner) key per D-02:
   - Use associative array: `declare -A SEEN`
   - Merge provenance: "health + todo" if same failure from multiple sources

10. Write FAILURES.md (D-05, D-19 through D-22):
    - Header with environment: Arch Linux, `nvim --version`, tool versions
    - Table with columns: ID, Description, Owner, Status, Repro Steps, Provenance
    - ID format: BUG-001, BUG-002, etc. (D-20)
    - Initial status: "Discovered" (D-24, D-25)
    - Owner mapping: keymap → core/keymaps/, plugin → plugin files, core → init.lua/core/*, tool → external (D-13 through D-16)

11. Exit 0 on success, non-zero on script failure.
  </action>
