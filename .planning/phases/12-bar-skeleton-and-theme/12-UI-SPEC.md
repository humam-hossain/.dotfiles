# Phase 12 — UI Design Contract

> Visual and interaction contract for Phase 12: Bar Skeleton and Theme.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> This phase delivers a visible, correctly themed Quickshell bar docked at the
> top of all screens. No functional widgets — only placeholder labels in pill
> containers to verify layout, font rendering, and three-section structure.

---

## Spacing Scale

Declared values (multiples of 4 only). Sourced from `.config/waybar/style.css`
exact pixel values (D-03 in CONTEXT.md).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Workspace button padding (`padding: 0 6px` rounds to 4px base unit); icon gaps |
| sm | 8px | Inter-module gap (`Row { spacing: 8 }` inside BarGroup; `margin: 6px 4px` = 4px each side) |
| md | 14px | Module horizontal padding (`padding: 6px 14px` — horizontal component) |
| lg | 24px | Reserved for future section-level spacing (phases 13–16) |
| xl | 32px | Reserved for layout-level gaps (phases 13–16) |

**Active in Phase 12:**

| Element | Value | Source |
|---------|-------|--------|
| ModulePill vertical padding | 6px | `padding: 6px 14px` from waybar style.css |
| ModulePill horizontal padding | 14px | `padding: 6px 14px` from waybar style.css |
| ModulePill border-radius | 8px | `border-radius: 8px` from waybar style.css |
| Inter-module spacing (BarGroup Row) | 8px | `margin: 6px 4px` (4px each side) from waybar style.css |
| Bar margin to screen edge | 0px | D-11: flush to screen top, no floating margin |
| Workspace button inner padding | 6px | `padding: 0 6px` from waybar style.css |
| Workspace button border-radius | 6px | `border-radius: 6px` from waybar style.css |

Exceptions: Bar height is content-driven, not fixed. `exclusiveZone: height`
(dynamic). No fixed bar height token declared. (Source: D-12, CONTEXT.md)

## Typography

Single font family across all bar elements. Sourced from `style.css` global
reset (D-03, CONTEXT.md).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Bar label (body) | 14px | Bold (700) | 1.0 (single-line bar elements) | All module text, placeholder labels |
| Icon glyphs | 14px | Bold (700) | 1.0 | Nerd Font glyph codepoints (phases 13–16) |

**Font declaration for QML:**

```qml
font.family: "JetBrainsMono Nerd Font"
font.pixelSize: 14
font.bold: true
```

**Rationale:** Bar labels are single-line UI elements in a status bar context.
Line-height 1.5 is not applicable. The Waybar reference uses no explicit
line-height — content height is driven by font metrics alone.

**Phase 12 placeholder text:** Static string literals `"Left"`, `"Center"`,
`"Right"` — same 14px bold rendering as future widget labels.

### Drop Shadow

Applied to the bar background Rectangle via Qt5Compat.GraphicalEffects DropShadow.
Mirrors Waybar's `box-shadow: 0 4px 6px rgba(0,0,0,0.3)` exactly (D-05, CONTEXT.md).

| Property | Value |
|----------|-------|
| `verticalOffset` | 4 |
| `horizontalOffset` | 0 |
| `radius` | 6 |
| `color` | `Qt.rgba(0, 0, 0, 0.3)` |
| `layer.enabled` | `true` (on the background Rectangle) |

### PanelWindow Background

`color: "transparent"` — the bar window itself is transparent. The black
background is rendered by the inner Rectangle (`Colours.barBg`), not the window.
This matches Waybar's `background: transparent` on `window#waybar` (D-04, CONTEXT.md).

## Copywriting Contract

Phase 12 has no user-facing copy in the conventional web sense. The bar is a
desktop UI shell component. The table below captures all visible text in Phase 12.

| Element | Copy | Notes |
|---------|------|-------|
| Left section placeholder | `"Left"` | Static Text label inside ModulePill; removed in Phase 13 |
| Center section placeholder | `"Center"` | Static Text label inside ModulePill; removed in Phase 13 |
| Right section placeholder | `"Right"` | Static Text label inside ModulePill; removed in Phase 13 |
| Install script relog reminder | `"Log out and back in for i2c group to take effect (required for ddcutil)"` | Printed to stdout by `arch/quickshell.sh` after usermod (D-16, CONTEXT.md) |

**Empty state:** Not applicable in Phase 12. Placeholder labels ensure sections
are always visible. A multi-monitor zero-screen scenario cannot occur — Quickshell
requires at least one connected monitor to start.

**Error state:** Not applicable in Phase 12. No runtime data fetching, no network
calls, no IPC queries. The bar is purely static layout with hardcoded text.

**Destructive actions:** None in Phase 12. The install script (`arch/quickshell.sh`)
performs system writes (`pacman -S`, `usermod`, `/etc/modules-load.d/`) but these
are idempotent CLI operations with no confirmation prompt — consistent with
`arch/waybar.sh` pattern and `set -euo pipefail` error handling.

## Component Inventory

Components authored in Phase 12. These are the shared foundational components
consumed by phases 13–16.

### `Colours.qml` — Theme Singleton

| Property | Contract |
|----------|----------|
| Declaration | `pragma Singleton` |
| Module path | `.config/quickshell/theme/Colours.qml` |
| Exposes | All 26 canonical Catppuccin Mocha hex values as `readonly property color` + 8 semantic aliases |
| Registration | Must be registered in `qmldir` inside `theme/` directory |

### `ModulePill.qml` — Shared Pill Wrapper

| Property | Contract |
|----------|----------|
| Module path | `.config/quickshell/ModulePill.qml` |
| Shape | `Rectangle { radius: 8; color: Colours.moduleBg }` |
| Padding | `topPadding: 6; bottomPadding: 6; leftPadding: 14; rightPadding: 14` |
| Content slot | `default property alias content: inner.children` (or equivalent default alias) |
| Usage in phases 13–16 | Every widget wraps itself in ModulePill — this is the single source of truth for pill shape |

### `BarGroup.qml` — Pill-Row Section Container

| Property | Contract |
|----------|----------|
| Module path | `.config/quickshell/BarGroup.qml` |
| Layout | `Row { id: row; spacing: 8 }` |
| Content slot | `default property alias children: row.children` |
| Usage | `BarGroup { ModulePill { Text { text: "Left" } } }` |

### `BarContent.qml` — PanelWindow Content Root

| Property | Contract |
|----------|----------|
| Module path | `.config/quickshell/BarContent.qml` |
| Layout | Three-section horizontal layout: left (left-aligned), center (horizontally centered), right (right-aligned) |
| Background | Full-width `Rectangle { color: Colours.barBg }` with DropShadow applied |
| Window transparency | PanelWindow `color: "transparent"` |
| Keyboard focus | `WlrKeyboardFocus.None` set unconditionally on PanelWindow (Architecture pattern) |
| Exclusive zone | `exclusiveZone: height` (content-driven) |
| Anchoring | `anchors { top: true; left: true; right: true }` — flush top, full width |

### `Bar.qml` — Multi-Monitor Scope

| Property | Contract |
|----------|----------|
| Module path | `.config/quickshell/Bar.qml` |
| Pattern | `Scope { Variants { model: Quickshell.screens; delegate: BarContent { screen: modelData } } }` |
| Monitor behavior | Identical bar on every connected screen; bars added/removed dynamically |

### `shell.qml` — Entry Point

| Property | Contract |
|----------|----------|
| Module path | `.config/quickshell/shell.qml` |
| Pattern | `Scope { Bar {} }` |

## Layout Contract

### Three-Section Structure

```
┌─────────────────────────────────────────────────────────┐  ← PanelWindow (transparent)
│ ████████████████████████████████████████████████████████│  ← Rectangle (barBg #000000)
│ [Left Pills]          [Center Pills]          [Right Pills]│
│  ↑ BarGroup left       ↑ BarGroup center       ↑ BarGroup right
└─────────────────────────────────────────────────────────┘
   ← exclusiveZone: height → tiling windows pushed below
```

### Phase 12 Placeholder State

```
┌─────────────────────────────────────────────────────────┐
│ [  Left  ]          [  Center  ]          [  Right  ]   │
│  #1e1e2e pill        #1e1e2e pill          #1e1e2e pill  │
│  14px bold           14px bold             14px bold     │
│  #cdd6f4 text        #cdd6f4 text          #cdd6f4 text  │
└─────────────────────────────────────────────────────────┘
  ↑ DropShadow: verticalOffset 4, radius 6, rgba(0,0,0,0.3)
```

## Interaction Contract

Phase 12 has no interactive elements. All three placeholder labels are static.
No hover effects, no click handlers, no transitions in this phase.

Hover color transitions (`Behavior on color { ColorAnimation { duration: 120 } }`)
are ANIM-01 in Phase 16 — explicitly deferred.
