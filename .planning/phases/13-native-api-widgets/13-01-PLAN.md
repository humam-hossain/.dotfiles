    ```qml
    pragma Singleton
    import Quickshell
    import Quickshell.Services.Pipewire
    import QtQuick

    Singleton {
        id: root

        readonly property var  defaultSink:    Pipewire.defaultAudioSink
        readonly property real volume:         defaultSink && defaultSink.audio ? defaultSink.audio.volume : 0
        readonly property bool muted:          defaultSink && defaultSink.audio ? defaultSink.audio.muted  : false
        readonly property int  volumePercent:  Math.round(volume * 100)
        readonly property string sinkName:     defaultSink ? (defaultSink.description || defaultSink.name || "Audio") : ""

        function setVolume(percent) {
            if (!defaultSink || !defaultSink.audio) return
            defaultSink.audio.volume = Math.max(0, Math.min(1, percent / 100))
        }
        function bumpVolume(delta) { setVolume(volumePercent + delta) }
        function toggleMute() {
            if (!defaultSink || !defaultSink.audio) return
            defaultSink.audio.muted = !defaultSink.audio.muted
        }

        PwObjectTracker {
            objects: root.defaultSink ? [root.defaultSink] : []
        }
    }
    ```

    Implementation notes:
    - `pragma Singleton` MUST be the first line (no leading imports). Imports follow: Quickshell, Quickshell.Services.Pipewire, QtQuick.

    - `PwObjectTracker` MUST live as a direct child of `Singleton { ... }` — never inside a Loader or Component (RESEARCH.md Pitfall 1).
    - `setVolume` clamps to 0..100 input range (security threat-model bound: prevents writing volume outside `[0, 1]` to `.audio.volume`).
    - The tracker's `objects` binding re-evaluates when `Pipewire.defaultAudioSink` changes (headphone plug/unplug) — re-binding is automatic via the QML reactive expression (D-05).

            .sort((a, b) => a.id - b.id)

                            delete next[id]

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted MPRIS players → MprisService | Third-party media apps publish artist/title/album strings via D-Bus; Quickshell receives them as `trackArtist`/`trackTitle`/`trackAlbum`. |
| PipeWire defaultAudioSink writes | `setVolume`/`toggleMute` write to `.audio.volume`/`.audio.muted` on the system audio sink object. |
| Hyprland rawEvent payload | `event.data` for `urgent` events comes from Hyprland IPC and is parsed as text. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-AUD-01 | Tampering | AudioService.setVolume | mitigate | Clamp input via `Math.max(0, Math.min(1, percent / 100))` BEFORE assigning to `.audio.volume`. Prevents out-of-range writes that could surprise PipeWire. Verified by acceptance criteria grep. |
| T-13-AUD-02 | Denial of Service | AudioService.bumpVolume rapid invocation | accept | PipeWire handles rapid writes natively (D-25). No client-side throttle needed. Risk: keyboard-held volume key spams writes — accepted because PipeWire's own coalescing covers this. |
| T-13-MPRIS-01 | Injection | MprisService.activePlayer string properties | mitigate | This service exposes the player object reference only; it never stringifies player metadata into shell commands. Plan 03 (MusicWidget) MUST follow the same rule — pass `trackArtist`/`trackTitle` ONLY into `Text { text: ... }` and `ToolTip.text`, never into `Process.command` or `Hyprland.dispatch`. |
| T-13-HYP-01 | Information Disclosure | HyprWorkspaces.urgentIds | accept | Workspace IDs are non-sensitive integers. No PII or capability tokens. |
| T-13-HYP-02 | Tampering | Hyprland.dispatch with user input | mitigate | This plan does NOT invoke `Hyprland.dispatch` — only the workspace widget (Plan 02) does, and it uses static string literals `"workspace e+1"` / `"workspace e-1"` only. Confirmed in Plan 02 acceptance criteria. |
| T-13-PW-01 | Spoofing | Pipewire.defaultAudioSink swap | accept | Trusting Quickshell's binding to the kernel-side default sink object. Re-binding on swap is part of the design (D-05). |
</threat_model>
