# Phase 13 — UI Design Contract

> Visual and interaction contract for the four native-API widgets (Workspaces, Volume, Music, Tray). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Note:** This is a Quickshell/QML desktop-shell phase, not a web app. shadcn / component-library tooling is not applicable. Visual primitives come from Phase 12 (`ModulePill`, `BarGroup`, `Colours.qml`); this contract locks the four widgets' visible surface against those primitives.

| Property | Value |
|----------|-------|
| Tool | none (native QML) |

## Spacing Scale

Declared values (multiples of 4, inherited from Phase 12 D-03):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Reserved (currently unused in Phase 13) |
| sm | 8px | `BarGroup` inter-widget spacing; recommended workspace-button gap inside the workspaces pill |
| md | 14px | `ModulePill` horizontal internal padding (14px L/R per Phase 12 D-03) |
| md-v | 6px | `ModulePill` vertical internal padding (6px T/B per Phase 12 D-03) |
| lg | 21px | Tray icon size — exact match to Waybar `tray.icon-size: 21` (D-39) |

Exceptions:
- `md` and `md-v` are 14/6 (not the canonical 16/8) because they are inherited from `ModulePill` whose values match the live Waybar reference. Phase 13 does NOT override them.
- `lg = 21px` exists ONLY as a tray-icon raster size (matches Waybar). It is not a layout-spacing token.

Per-widget spacing rules:

| Widget | Internal gap | Source |
|--------|--------------|--------|
| WorkspacesWidget (between workspace buttons inside the pill) | 8px | matches `BarGroup.spacing` (Phase 12 D-09) for visual cohesion |
| VolumeWidget (icon ↔ percentage text) | 4px | tight pairing reads as one unit |
| MusicWidget (icon ↔ "artist - title" text) | 4px | tight pairing reads as one unit |
| TrayWidget (between tray icons inside the pill) | 8px | matches Waybar `tray.spacing: 10` rounded down to the project's 4-multiple grid |

## Typography

Phase 13 inherits one type style from Phase 12 (D-03) and declares one supporting role for tooltips. No new heading/display tiers — the bar surface has no headings.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Module text | 14px | bold (700) | 1.2 (content-driven) | All widget labels: workspace icons, volume %, music "artist - title", urgent fallback glyphs |
| Tooltip | 14px | regular (400) | 1.4 | `ToolTip.text` on VolumeWidget and MusicWidget (D-27, D-34) |

Font family for ALL roles: `JetBrainsMono Nerd Font`.

Glyph contract — exact codepoints to use:

| Widget | State | Glyph | Codepoint |
|--------|-------|-------|-----------|
| WorkspacesWidget | active workspace | `` | U+F444 (Nerd Font Material `circle`) |
| WorkspacesWidget | default workspace | `` | U+F4C3 (Nerd Font Material `circle-outline`) |
| VolumeWidget | muted OR `volume == 0%` | `` | U+F026 (Font Awesome `volume-off`) |
| VolumeWidget | `0% < volume < 33%` | `` | U+F027 (Font Awesome `volume-down`) |
| VolumeWidget | `33% <= volume < 66%` | `` | U+F027 (same low-volume glyph; planner may swap to `` U+FA7D if a distinct mid glyph is desired — researcher confirms the existing Waybar set uses two icons "" and "" only, so we match that) |
| VolumeWidget | `volume >= 66%` | `` | U+F028 (Font Awesome `volume-up`) |
| MusicWidget | always (D-29) | `` | U+F1BC (Font Awesome `spotify` glyph used in current Waybar `custom/music`) |
| MusicWidget | player exists, no track metadata | `` (same icon) + literal text `No track` | D-33 |
| TrayWidget | icon-load failure fallback | `` | U+F128 (question-mark) per D-44 |

> Truncation: MusicWidget uses `Text.elide: Text.ElideRight` with `Layout.maximumWidth` derived from `30 * fontMetrics.averageCharacterWidth` (D-30). Workspaces/Volume/Tray have no text truncation (content-driven width).

## Color

The bar uses the Catppuccin Mocha palette via `Colours.qml` semantic aliases. Phase 13 introduces NO new tokens. The 60/30/10 framing is adapted to a desktop-shell context where the "page background" is the user's wallpaper (transparent panel) and the bar is the dark surface.

| Role | Token (Colours.qml alias) | Hex | Usage |
|------|---------------------------|-----|-------|
| Dominant (60%) | `Colours.barBg` | `#000000` | Bar strip background (Phase 12 D-02 — surface0 override) |
| Secondary (30%) | `Colours.moduleBg` | `#1e1e2e` | Pill background for every widget (Phase 12 D-06) |
| Accent (10%) | `Colours.accent` (= mauve) | `#cba6f7` | Active workspace ONLY |
| Body text | `Colours.textColor` | `#cdd6f4` | Volume %, music text, occupied workspaces, tray fallback glyph |
| Muted text | `Colours.subtextColor` | `#bac2de` | Empty workspaces (workspaces with no toplevels) |
| Destructive / Alert | `Colours.critical` (= red) | `#f38ba8` | Urgent workspace icon tint; NeedsAttention tray icon tint |
| Warning (reserved) | `Colours.warning` (= yellow) | `#f9e2af` | Not used in Phase 13 (reserved for Phase 14 CPU/memory thresholds) |
| Success (reserved) | `Colours.success` (= green) | `#a6e3a1` | Not used in Phase 13 |

**Accent reserved for:** active workspace icon (`ws.active === true`) — exactly one element per bar at any moment. Volume, music, and tray do NOT use accent — they use `textColor` for their primary glyphs to preserve the "accent = active workspace" semantic.

**Critical reserved for:** urgent workspace icon override AND tray items in `NeedsAttention` SNI status (D-14, D-41) — both are involuntary attention signals, not user-chosen states.

**Disabled / dimmed states (opacity, not color):**

| State | Opacity | Source |
|-------|---------|--------|
| VolumeWidget muted (text portion) | 60% | D-22 |
| MusicWidget when `player.canControl === false` | 60% | D-32 |
| All other states | 100% | — |

**Hover colors:** Phase 13 ships static color states ONLY. There are no hover color targets in this phase (D-50). Hover transitions and target colors are deferred to Phase 16 (ANIM-01). The base/default state colors specified above are the only colors rendered.

## Copywriting Contract

Phase 13 is a desktop bar with minimal copy. Most widgets are icon-only or icon-plus-data; the only freeform copy is in tooltips and the music fallback string.

| Element | Copy |
|---------|------|
| Primary CTA (per widget) | None — widgets are direct manipulators (click = activate / launch / toggle). No buttons with verb labels. |
| Empty state — Workspaces | (none rendered — list is empty for ~1 frame on cold start, then populates; D-19) |
| Empty state — Volume | Widget is hidden via `visible: false` when no default sink (D-28). No empty-state copy. |
| Empty state — Music (no player) | Widget is hidden via `visible: false` when `MprisService.activePlayer === null` (D-35). No empty-state copy. |
| Empty state — Music (player active, no metadata) | ` No track` (icon + literal text "No track"; D-33) |
| Empty state — Tray | Pill is hidden via `visible: SystemTray.items.length > 0` when tray is empty (D-40). No empty-state copy. |
| Error state — pavucontrol launch failure | Console warning only: `console.warn("pavucontrol exited with code " + code)` per D-48. No user-visible toast (Phase 13 has no toast surface). |
| Error state — Tray icon load failure | Render Nerd Font glyph `` (U+F128) as fallback Text (D-44). No accompanying message. |
| Tooltip — Volume | `"Default sink: <sink.description> — <percent>%"` (e.g. `"Default sink: HDA Intel PCH — 75%"`). When muted: append `" (muted)"`. (D-27) |
| Tooltip — Music | Three lines: `"<trackArtist>\n<trackTitle>\n<trackAlbum>"`. Lines for empty fields are omitted (no leading/trailing blank lines). (D-34) |
| Destructive confirmation | None — Phase 13 has no destructive actions. (Right-click mute is reversible; left-click pavucontrol is non-destructive; click play/pause is reversible; tray right-click opens app's own native menu.) |

Voice and tone:
- Tooltip strings are diagnostic, not friendly: `"Default sink: HDA Intel PCH — 75%"`, not `"You're listening on HDA Intel PCH at 75%"`.
- All copy is English, sentence case, no terminal punctuation.

## Interaction Contract

Phase 13 introduces interactions on every widget. Locked here so the executor cannot reinterpret the gesture map.

| Widget | Gesture | Effect | Decision Source |
|--------|---------|--------|-----------------|
| WorkspacesWidget — workspace button | Left-click | `workspace.activate()` (cross-monitor capable) | D-16 |
| WorkspacesWidget — pill area | Wheel down | `Hyprland.dispatch("workspace e+1")` (next, wraps) | D-18 |
| WorkspacesWidget — pill area | Wheel up | `Hyprland.dispatch("workspace e-1")` (previous, wraps) | D-18 |
| VolumeWidget — pill area | Left-click | Launch `pavucontrol` via `Process.startDetached()` | D-23 |
| VolumeWidget — pill area | Right-click | `AudioService.toggleMute()` | D-24 |
| VolumeWidget — pill area | Wheel | `bumpVolume(±5)` per detent (sign of `wheel.angleDelta.y`) | D-25 |
| MusicWidget — pill area | Left-click | `player.togglePlaying()` (works in Stopped state too) | D-31 + RESEARCH A2 correction |
| MusicWidget — pill area | (when `!canControl`) | `MouseArea.enabled = false`, opacity 0.6 | D-32 |
| TrayWidget — single icon | Left-click | `item.activate(globalX, globalY)` | D-42 |
| TrayWidget — single icon | Right-click | Open `QsMenuAnchor` against `item.menu`; dismiss via `HyprlandFocusGrab` | D-42, D-43 |
| ALL clickable surfaces | Hover | `MouseArea.cursorShape: Qt.PointingHandCursor` (cursor only — no color change in Phase 13) | D-45, D-50 |
| ALL widget hit areas | — | Full pill via `MouseArea { anchors.fill: parent }` (Fitts' law) | D-46 |

**Tray context-menu placement:** anchored below the bar (`edges: Edges.Top` of the menu attached to the screen-bottom of the clicked icon), top-aligned to the clicked icon's horizontal position. Menu opens downward into the workspace area. Outside-click dismiss via `HyprlandFocusGrab`; never `grabFocus: true` (Phase 12 P-16).

## State Inventory (per widget)

Locks the visual states the checker and executor must support. Each row is a state the implementation must render correctly.

### WorkspacesWidget

| State | Visual |
|-------|--------|
| Active workspace | Glyph ``, color `Colours.accent` (mauve `#cba6f7`) |
| Occupied workspace (has toplevels, not active) | Glyph ``, color `Colours.textColor` (`#cdd6f4`) |
| Empty workspace (no toplevels, not active) | Glyph ``, color `Colours.subtextColor` (`#bac2de`) |
| Urgent workspace | Glyph color override to `Colours.critical` (`#f38ba8`); preserves the active/default glyph |
| Empty list (initial paint) | Pill renders with no buttons inside (~1 frame); no placeholder copy (D-19) |

### VolumeWidget

| State | Visual |
|-------|--------|
| Active, ≥66% volume | ` 75%` — icon `` + percent text in `Colours.textColor` |
| Active, 33–65% | ` 50%` — icon `` + percent text |
| Active, 1–32% | ` 15%` — icon `` + percent text |
| Active, 0% (not muted) | ` 0%` — muted icon per D-21 (intentional deviation: 0% renders as muted) |
| Muted | ` 75%` — muted icon AND text opacity 60% (D-22) |
| No default sink | Hidden (`visible: false`, D-28) |

### MusicWidget

| State | Visual |
|-------|--------|
| Player active, has metadata | ` Pink Floyd - Money` (≤30 chars; right-elided to `…` if longer) |
| Player active, no metadata | ` No track` (D-33) |
| Player active, `!canControl` | Same content as above, opacity 60%, MouseArea disabled (D-32) |
| No player | Hidden (`visible: false`, D-35) |

### TrayWidget

| State | Visual |
|-------|--------|
| 1+ tray items, all Active/Passive | Pill renders icons at 21px, original colors (no tint) |
| Tray item with `NeedsAttention` status | Icon tinted `Colours.critical` (D-41) |
| Tray icon URI fails to load | Fallback Nerd Font `` glyph as Text in `Colours.textColor` (D-44) |
| 0 tray items | Pill hidden (`visible: false`, D-40) |
| Right-click | `QsMenuAnchor` opens below bar; outside-click dismisses via `HyprlandFocusGrab` |

## Multi-Monitor Behavior

Inherited from Phase 12 D-13 — `Variants { model: Quickshell.screens }` renders one bar per screen.

| Concern | Behavior |
|---------|----------|
| Workspaces — list source | Global `Hyprland.workspaces.values` (D-17) — every bar shows the SAME list |
| Workspaces — active highlight | Each bar highlights `ws.active` (per-monitor visible workspace) (D-12). On dual-monitor, two different workspaces are mauve simultaneously, one per bar. |
| Volume — sink source | Global `Pipewire.defaultAudioSink` — all bars show the same percentage |
| Music — player source | Global `MprisService.activePlayer` — all bars show the same track |
| Tray — items source | Global `SystemTray.items` — all bars show the same icons |

The visual is deliberately uniform across screens; only the workspaces' `active` highlight differs per bar.
