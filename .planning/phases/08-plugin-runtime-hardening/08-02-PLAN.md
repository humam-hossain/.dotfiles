<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Neovim events -> repo autocmd callbacks | Untrusted buffer/runtime state crosses into autosave, format, and LSP attach code |
| repo open helper -> OS opener | External command selection and failures come from the host environment, not trusted repo code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | D | `core/keymaps.lua` | mitigate | Refuse autosave on special, unnamed, or non-modifiable buffers so FocusLost cannot crash or mutate unsafe buffers. |
| T-08-06 | D | `plugins/conform.lua` | mitigate | Keep format-on-save behind explicit buffer/filetype guards so picker, fugitive, and nofile buffers do not enter formatter code paths. |
| T-08-07 | I | `core/open.lua` | mitigate | Surface `vim.ui.open()`'s returned error string so the user sees the real failure instead of a misleading generic notification. |
| T-08-08 | D | `plugins/lsp.lua` | mitigate | Gate attach/highlight setup on valid client and buffer checks to prevent attach-time crashes in special or unsupported buffers. |
</threat_model>
