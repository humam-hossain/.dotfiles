<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-selected file path -> external open helper | Untrusted path or URL text crosses from editor state into OS default-app dispatch |
| neo-tree node path -> helper | File explorer selection can contain spaces and special characters that must not be shell-interpreted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | T | `.config/nvim/lua/core/open.lua` | mitigate | Use `vim.ui.open()` directly instead of shell-string commands so paths are not interpolated through a shell. |
| T-01-02 | D | `.config/nvim/lua/core/open.lua` | mitigate | Emit `vim.notify(..., ERROR)` when open fails so the user gets an explicit failure instead of silent breakage. |
| T-01-03 | T | `.config/nvim/lua/plugins/neotree.lua` | mitigate | Route neo-tree actions through the shared helper so plugin code cannot drift back to `jobstart` with Linux-only commands. |
</threat_model>
