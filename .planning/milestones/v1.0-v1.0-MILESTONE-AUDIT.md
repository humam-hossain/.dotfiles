## Integration Checker Findings

### CRITICAL (require fixes before completion)

#### 1. `whichkey.register()` never called — KEY-01, KEY-02

`whichkey.lua` exports `M.register()` which reads `registry.groups` and would register 8 domain groups into which-key. This function is **never called** from any file. `misc.lua` runs `which-key.setup({})` with no registry integration.

**Impact:** Users see no domain taxonomy in which-key popups. KEY-01 registry-to-which-key leg severed. KEY-02 domain taxonomy exists in code but never exposed.

**Fix:** Add `require("core.keymaps.whichkey").register()` after `which-key.setup({})` in `misc.lua`.

#### 2. `get_by_scope("plugin_local")` scope key mismatch — KEY-03

`attach.lua` calls `registry.get_by_scope("plugin_local")` (underscore). `registry.get_by_scope()` only recognizes `"plugin-local"` (hyphen). Always returns `{}`. `apply_neotree()` silently fails. Neither `apply_neotree()` nor `get_plugin_local_maps()` is called anywhere anyway.

**Impact:** 9 neo-tree plugin-local registry entries are unreachable. Live neotree window mappings (`S`, `s`, `t`, `w`, etc.) come from `neotree.lua`'s hardcoded `window.mappings` — dual source of truth violating KEY-03.

**Fix:** Fix scope string to `"plugin-local"` in `attach.lua` AND call `apply_neotree(bufnr)` from neotree setup. Or: remove orphaned `plugin_local` registry entries and document that `neotree.lua window.mappings` is the intentional single source.

#### 3. Domain-mismatched neo-tree keys registered under snacks.lua — KEY-01, TOOL-02

`neotree.lua` uses `explorer_keys()` which only returns entries with `domain == "e"`. Three keys with other domains land in `snacks.lua`'s `get_all_keys()`:
- `<leader>gc` (domain `"g"`) — git commits via neo-tree
- `<leader>ngs` (domain `"g"`) — neo-tree git status
- `<leader>nb` (domain `"b"`) — neo-tree buffers

When pressed, lazy.nvim loads **snacks**, not neo-tree. `vim.cmd(":Neotree ...")` then fails silently if neo-tree not yet loaded.

**Fix:** Add `registry_keys_for("nvim-neo-tree/neo-tree.nvim")` helper, or change `neotree.lua` to use `get_all_keys()` filtered by plugin name.

### WARNING

#### 4. noice.nvim still active — UX-01

`misc.lua` loads `noice.nvim` with `cmdline = { enabled = true }`. VERIFICATION evidence checked for `notify.lua` file absence, not noice plugin presence. UX-01 claim "snacks replacing noice" is partially false.

**Fix:** Either remove noice from `misc.lua` and use snacks for cmdline UI, or update UX-01 wording to reflect intentional partial retention.
